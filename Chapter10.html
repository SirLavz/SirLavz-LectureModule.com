üìò Key Concepts

Transaction: A logical unit of work that must be either fully completed or aborted. It ensures the database transitions from one consistent state to another.

ACID Properties:

  • Atomicity: All operations complete or none do
  • Consistency: Maintains database integrity
  • Isolation: Transactions do not interfere
  • Durability: Committed changes persist

üîê Concurrency Control

Ensures serializability in multiuser environments. Prevents problems like:

  • Lost Updates: Overwriting changes from concurrent transactions
  • Uncommitted Data: Reading data from rolled-back transactions
  • Inconsistent Retrievals: Reading partially updated data

üîí Locking Mechanisms

  • Binary Lock: Locked or unlocked
  • Shared Lock: Read-only access
  • Exclusive Lock: Write access
  • Lock Granularity: Database, table, page, row, field

Two-Phase Locking (2PL):

  • Growing Phase: Acquire all locks
  • Shrinking Phase: Release locks

‚è±Ô∏è Time Stamping & Optimistic Control

  • Time Stamping: Assigns unique timestamps to transactions
  • Optimistic Control: Assumes minimal conflict; validates before commit
  • Wait/Die & Wound/Wait: Deadlock prevention strategies

üßØ Recovery Management

Restores database to a consistent state after failure. Uses:

  • Transaction Log: Records all changes
  • Deferred Write: Updates after commit
  • Write-Through: Updates immediately
  • Checkpoints: Save consistent states periodically
