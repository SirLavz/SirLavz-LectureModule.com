ðŸ“˜ Key Concepts

Relation: A table composed of rows (tuples) and columns (attributes). Each relation represents an entity set.

Keys:

  • Primary Key: Uniquely identifies each row
  • Foreign Key: Refers to a primary key in another table
  • Candidate Key: Minimal superkey
  • Composite Key: Made of multiple attributes

Integrity Rules:

  • Entity Integrity: No nulls in primary key
  • Referential Integrity: Foreign key must match a valid primary key or be null

ðŸ”— Relationships in Relational Model

  • 1:1: One entity relates to one other entity
  • 1:M: One entity relates to many others (most common)
  • M:N: Many-to-many; resolved via a bridge table

Example: A Student enrolls in many Courses; each Course has many Students. Use an Enrollment table to resolve M:N.
