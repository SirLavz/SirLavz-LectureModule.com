DBMS Architecture Guide

DBMS Architecture is the structural design of a database management system that defines how users interact with the system, how data is processed, and how it is stored and retrieved. It helps manage data efficiently and ensures scalability, security, and performance.

  • 1-Tier Architecture: All components on a single machine.
  • 2-Tier Architecture: Client directly connects to the database server.
  • 3-Tier Architecture: Client → Application Server → Database Server.

1-Tier

Used in standalone applications like MS Access. Simple and local.

2-Tier

Client application communicates with the database server directly. Common in LAN setups.

3-Tier

Modern architecture for web/mobile apps. Involves a client, application server, and database server.

  • Client: ReactJS frontend
  • App Server: Node.js or Java Spring Boot
  • Database: PostgreSQL/MySQL

This setup ensures scalability, performance, and security in a real-world airline reservation system.

Feature 1-Tier 2-Tier 3-Tier

  • DBMS Architecture defines how components of a database interact.
  • 1-Tier is local, 2-Tier is for LAN apps, 3-Tier is for scalable web apps.
  • 3-Tier architecture is the most widely used in real-world enterprise systems.
