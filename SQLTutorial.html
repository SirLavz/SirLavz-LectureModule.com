The SQL Encyclopedia

A complete 30-day technical curriculum for Laguna University students.

Stage 1: The Foundations

Day 1-3 Data Retrieval Basics

Every query starts with SELECT. We learn to filter rows using WHERE. Note: SQL processes FROM before WHERE, and WHERE before SELECT.

SELECT name, age FROM students WHERE age >= 18;

Day 4-7 Logic & Organization

We use LIKE for pattern matching (e.g., 'A%' finds all names starting with A). Sorting is handled by ORDER BY, and LIMIT prevents data overload.

Stage 2: Relationships & Aggregates

Day 8-10 The Mechanics of Joins

Joins connect tables via Primary and Foreign Keys. INNER JOIN returns only matching rows; LEFT JOIN returns everything from the first table plus matches.

[attachment_0](attachment) SELECT orders.id, customers.name FROM orders
JOIN customers ON orders.cid = customers.id;

Day 11-15 Aggregation Logic

Aggregates (SUM, AVG) collapse rows. Remember: Any non-aggregated column in your SELECT must appear in your GROUP BY clause.

Stage 3: Advanced Analysis

Day 16-17 Clean Logic & CTEs

CASE adds If/Then logic. CTEs (WITH) allow you to break complex queries into readable, temporary blocks.

Day 18-21 Window Functions & Sets

Window functions (OVER) perform calculations across rows while keeping individual records visible. Essential for Running Totals and Ranking.

SELECT name, RANK() OVER(PARTITION BY dept ORDER BY salary DESC) FROM staff;

Stage 4: Architecture & Design

Day 22-24 Database Normalization

Organizing data to minimize redundancy. We move from 1NF (atomic values) to 2NF (partial dependency) to 3NF (transitive dependency).

Day 25-27 Safety, Procedures, & Security

Transactions ensure data safety (ACID). Stored Procedures automate scripts. DCL (GRANT/REVOKE) secures access.

Day 28-30 Enterprise SQL

Understanding OLAP for Big Data and how to prep SQL data for Visualization tools like Tableau or Power BI.
