🛒 DBMS Architecture Activity

📘 Application: Shopee / Lazada (E-Commerce)

An example activity output using a three-tier database architecture for an e-commerce platform like Shopee or Lazada. Students should analyze how data flows through each layer.

🔍 Layer Descriptions with Real Use-Case

1. Presentation Layer

Interface accessed by users through mobile apps or browsers. E.g., a customer opens the Shopee app to search for “wireless earbuds”.

2. Application Layer

Processes business logic like search filters, cart handling, and payments. It handles the user request and decides what to send to or fetch from the database.

3. Database Layer

Stores data like users, products, orders, and payment logs. For instance, orders are inserted and inventory is updated when a purchase is made.

🔁 Example Flow: Buying an Item
  • User taps "Add to Cart" for a phone case on the Shopee app.
  • The app sends a request to the application server.
  • Application layer checks item availability and creates a new order.
  • Database updates inventory, stores order, and logs payment.
✅ Justification for Using 3-Tier Architecture
  • Scalability: Layers can be scaled independently.
  • Security: The database is only accessible via the app layer.
  • Maintainability: Easy to modify UI or backend logic separately.
