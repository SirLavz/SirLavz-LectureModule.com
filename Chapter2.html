üìò Key Concepts

Data Model: An abstraction that organizes data elements and their relationships. It simplifies complex real-world environments.

Building Blocks:

  • Entity: Object or concept about which data is stored
  • Attribute: Property of an entity
  • Relationship: Association between entities
  • Constraint: Rule that restricts data values or relationships

Business Rules: Organizational policies that define how data is captured, processed, and related. They guide model creation.

üìä Major Data Models

ModelDescriptionStrengths
Hierarchical Tree-like structure with parent-child relationships Fast access, simple structure
Network Graph-like with multiple parent-child links Flexible relationships, better performance
Relational Tables with rows and columns, keys define relationships Logical simplicity, SQL support
Object-Oriented Data and behavior encapsulated in objects Supports complex data types, inheritance

üèóÔ∏è Three-Schema Architecture

  • External Schema: User-specific views
  • Conceptual Schema: Logical structure of the entire database
  • Internal Schema: Physical storage details

Data Independence:

  • Logical Independence: Change conceptual schema without affecting external views
  • Physical Independence: Change internal schema without affecting conceptual schema
