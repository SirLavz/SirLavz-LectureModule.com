📘 Key Concepts

Extended Entity Relationship (EER) Model: Enhances the basic ER model with semantic constructs like supertypes, subtypes, specialization, generalization, and inheritance.

Supertypes and Subtypes:

  • Supertype: A generic entity type with common attributes
  • Subtype: A specialized entity with unique attributes
  • Connected via “IS-A” relationships

Specialization: Top-down process of identifying subtypes from a supertype

Generalization: Bottom-up process of combining subtypes into a supertype

Inheritance: Subtypes inherit attributes and relationships from supertypes

🧩 EER Constructs

  • Subtype Discriminator: Attribute in supertype that determines subtype membership
  • Disjoint Subtypes: Each instance belongs to only one subtype
  • Overlapping Subtypes: Instances may belong to multiple subtypes
  • Completeness Constraint: Specifies whether all supertype instances must belong to a subtype (total vs partial)

📦 Entity Clustering

Used to simplify complex ER diagrams by grouping related entities and relationships into a virtual cluster. Helps reduce diagram clutter and improve readability during design.

🧠 Design Cases

  • 1:1 Relationships: Place foreign key in one entity, not both
  • Time-Variant Data: Create a new entity to track historical changes
  • Fan Traps: Occur when relationships are improperly modeled; resolved by restructuring relationships

🧪 Laboratory Activity: Modeling Supertypes and Subtypes

💡 Objectives
  • Apply EER modeling principles to a real-world scenario
  • Design supertypes and subtypes with appropriate constraints
  • Draw an EER diagram using Crow’s Foot notation
📋 Instructions
  1. Choose a domain (e.g., Employee roles, Vehicle types, Academic programs)
  2. Identify common attributes for the supertype
  3. Define subtypes with unique attributes
  4. Specify disjoint/overlapping and completeness constraints
  5. Draw the EER diagram using a tool like draw.io or Lucidchart
  6. Submit the diagram with a brief explanation of design decisions
