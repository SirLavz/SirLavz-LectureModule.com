1NF ‚Äì First Normal Form

Definition: A relation is in 1NF if every attribute contains only atomic (indivisible) values and there are no repeating groups.

Example 1: Multiple Phone Numbers

StudentPhone Numbers
Alice123-456; 789-012
Bob555-123

  • Solution: Split into separate rows:
    • StudentPhone(Student, Phone)
    • Rows: (Alice,123-456), (Alice,789-012), (Bob,555-123)

Example 2: Composite Address

  • Solution: Decompose Address into atomic columns:

Example 3: Multiple Order Items

O100Pen, Notebook, Eraser
O101Pencil

2NF ‚Äì Second Normal Form

A relation is in 2NF if it is in 1NF and every non-key attribute is fully functionally dependent on the entire primary key.

Example Table

StudentIDCourseIDCourseName
101C01Database
101C02Networks
  • PK = (StudentID, CourseID)
  • Partial dependency: CourseID ‚Üí CourseName
  • Violates 2NF because CourseName depends on part of the key.
  • Decompose into:
    • Enrollment(StudentID, CourseID)
    • Course(CourseID, CourseName)

3NF ‚Äì Third Normal Form

A relation is in 3NF if it is in 2NF and no transitive dependencies exist among non-key attributes.

Example Table

EmpIDEmpNameDeptIDDeptName
1AliceD01IT
2BobD02HR
  • PK = EmpID
  • Transitive dependency: EmpID ‚Üí DeptID ‚Üí DeptName
  • Violates 3NF because DeptName depends on DeptID, not directly on EmpID.
  • Decompose into:
    • Employee(EmpID, EmpName, DeptID)
    • Department(DeptID, DeptName)

BCNF ‚Äì Boyce-Codd Normal Form

Every determinant must be a candidate key. Stronger than 3NF.

Example Table

ProfessorCourseClassroom
Dr. SmithDBMSR101
Dr. SmithData MiningR101
Dr. LeeOperating SystemsR102
  • FDs: {Professor,Course} ‚Üí Classroom & Professor ‚Üí Classroom
  • Violates BCNF: Professor (non-key) determines Classroom.
  • Decompose into:
    • ProfessorRoom(Professor, Classroom)
    • ProfessorCourse(Professor, Course)

4NF ‚Äì Fourth Normal Form

In BCNF and no multi-valued dependencies exist.

Example Table

StudentHobbyLanguage
AlicePaintingEnglish
AlicePaintingFrench
AliceDancingEnglish
AliceDancingFrench
  • Student ‚Üí‚Üí Hobby & Student ‚Üí‚Üí Language
  • Violates 4NF: two independent multi-valued facts.
  • Decompose into:
    • StudentHobby(Student, Hobby)
    • StudentLanguage(Student, Language)

5NF ‚Äì Fifth Normal Form

Every join dependency is implied by candidate keys.

Example Table

SupplierPartProject
S1P1J1
S1P1J2
S1P2J1
S1P2J2
  • Implicit join dependency among (Supplier,Part,Project).
  • Violates 5NF: cannot losslessly reconstruct from pairwise joins.
  • Decompose into:
    • SupplierPart(Supplier, Part)
    • SupplierProject(Supplier, Project)
    • PartProject(Part, Project)

6NF ‚Äì Sixth Normal Form

Handles non-trivial join dependencies in temporal data.

Example Table

ContractIDClauseEffectiveDate
C01A2025-01-01
C01A2025-03-01
C01B2025-01-01
  • Clause has multiple EffectiveDates.
  • Violates 6NF if residual join dependencies exist.
  • Decompose into:
    • ContractClause(ContractID, Clause)
    • ClauseEffectiveDate(Clause, EffectiveDate)
