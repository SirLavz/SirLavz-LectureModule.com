ğŸ“˜ Key Concepts

Normalization: A process that organizes data to reduce redundancy and improve integrity. It involves decomposing tables based on dependencies.

Data Anomalies:

  • Update Anomaly: Inconsistent updates across redundant data
  • Insert Anomaly: Inability to add data due to missing related data
  • Delete Anomaly: Unintended loss of data when deleting related records

Functional Dependency: A relationship where one attribute uniquely determines another.

ğŸ“Š Normal Forms

Normal FormDescriptionRequirement
1NF Eliminates repeating groups Atomic values, unique rows
2NF Removes partial dependencies All non-key attributes fully dependent on primary key
3NF Removes transitive dependencies Non-key attributes depend only on the key
BCNF Stronger version of 3NF Every determinant is a candidate key

Sometimes, for performance reasons, designers intentionally introduce redundancy. This is called denormalization. It improves query speed but may increase storage and risk of anomalies.
