ðŸ“˜ Key Concepts

Performance Tuning: Activities aimed at reducing query response time and resource usage. It involves optimizing both client-side SQL and server-side DBMS configurations.

Query Processing Phases:

  1. Parsing: SQL is validated and transformed into an execution plan
  2. Execution: DBMS performs I/O operations and applies logic
  3. Fetching: Results are retrieved and returned to the client

ðŸ“Š Indexing & Optimization

  • B-tree Index: Balanced tree structure for range queries
  • Bitmap Index: Efficient for low-cardinality columns
  • Hash Index: Fast access for exact matches

Index Selectivity: Measures uniqueness of indexed values. High selectivity improves performance.

ðŸ§  Optimizer Strategies

  • Rule-Based Optimizer: Uses predefined rules to choose execution plans
  • Cost-Based Optimizer: Evaluates multiple plans based on statistics and resource cost
  • Optimizer Hints: Manual suggestions embedded in SQL to influence optimizer decisions

âœ… SQL Performance Tips

  • Use indexed columns in WHERE, ORDER BY, GROUP BY
  • Avoid SELECT *
  • Use literals and equality comparisons
  • Write AND conditions with most selective filters first
  • Minimize use of NOT and OR
