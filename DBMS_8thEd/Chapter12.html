ğŸ“˜ Key Concepts

DDBMS: A Distributed Database Management System governs storage and processing of logically related data across multiple sites connected via a network.

Distributed Processing: Logical operations are shared across independent systems.

Distributed Database: Data is stored across multiple physical locations but appears unified to users.

ğŸ§© Transparency Features

  • Distribution Transparency: Users access data without knowing its physical location
  • Transaction Transparency: Ensures consistency across distributed transactions
  • Failure Transparency: System recovers from site failures without user impact
  • Performance Transparency: Optimizes queries across sites
  • Heterogeneity Transparency: Supports different DBMSs and platforms

ğŸ“¦ Data Distribution Strategies

  • Fragmentation: Divides tables into subsets (horizontal, vertical, mixed)
  • Replication: Stores copies of data at multiple sites
  • Allocation: Determines where fragments and replicas reside

Distributed query optimization minimizes total cost using access time, communication cost, and CPU time. Techniques include:

  • Static vs. dynamic optimization
  • Rule-based vs. cost-based algorithms
  • Replica transparency to hide multiple copies

ğŸ” Concurrency & Two-Phase Commit

  • Concurrency Control: Prevents inconsistencies in multi-site transactions
  • Two-Phase Commit: Ensures all sites commit before final COMMIT
  • DO-UNDO-REDO Protocol: Supports recovery and consistency
